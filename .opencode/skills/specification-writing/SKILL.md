# Specification Writing Skill

## Process

1. Read `specs/observed/`.
2. Extract confirmed UI elements and behaviors.
3. Separate confirmed facts, assumptions, and open questions.
4. Write screen specs under `specs/screens/`.
5. Write flow specs under `specs/flows/` when an operation spans multiple steps.
6. Include testable acceptance criteria.
